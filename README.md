# OpenClaw Project Notes

Shared project notes for OpenClaw.

- Put one Markdown file per project in `projects/`
- Keep durable project context here so OpenClaw can index and search it
- Suggested filename format: `project-name.md`

Examples:
- `projects/openclaw.md`
- `projects/my-startup.md`
- `projects/home-automation.md`

## Convention

Default rule for OpenClaw:

- Notes-first work goes here.
- Use one Markdown file per project in `projects/`.
- Real code/build/deploy projects may get their own separate git repo when requested or clearly needed.
- Even when a project has its own repo, keep a short summary note here so it remains searchable in one place.

## Suggested workflow

1. Check whether a project note already exists in `projects/`.
2. If yes, update it.
3. If no, create a new file from `templates/project-template.md`.
4. Only create a separate code repo when the work is actually code-heavy or Lukas asks for one.
